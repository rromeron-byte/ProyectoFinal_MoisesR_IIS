## **¿Qué es Markdown?**



Markdown es un lenguaje de marcado ligero desarrollado por John Gruber y Aaron Swartz en 2004. Su intención principal es que el texto escrito sea fácil de leer y de escribir, tanto en su forma original como una vez convertido a otros formatos. Utiliza una sintaxis muy simple —como asteriscos para resaltar texto o guiones para crear listas— que luego puede transformarse en HTML u otros tipos de salida. Esto permite dar formato al contenido (títulos, listas, enlaces, etc.) sin la complejidad de usar directamente HTML o editores más pesados.



#### **Uso en Proyectos de Software**



En el ámbito del desarrollo, Markdown se ha convertido en un estándar para documentar proyectos debido a su sencillez y compatibilidad.



#### **Archivos README**



La mayoría de los repositorios incluyen un archivo README.md, que suele ser lo primero que ve cualquier persona interesada en el proyecto. Ahí se explica de qué trata, cómo instalarlo y cómo empezar a utilizarlo.



#### **Documentación General**



Markdown también se usa para otros documentos importantes como CONTRIBUTING.md, archivos de licencia, guías de estilo o documentación interna.

#### 

#### **Sistemas de Control de Versiones**



Plataformas como GitHub, GitLab y Bitbucket muestran los archivos Markdown con formato automáticamente, lo que facilita su lectura sin necesidad de pasos adicionales.



#### **Markdown y GitHub**

Relación, Ventajas y Desventajas



GitHub emplea una variante ampliada llamada GitHub Flavored Markdown (GFM), que incorpora funciones adicionales como tablas, casillas de verificación, resaltado de código y referencias a usuarios o issues. Gracias a esto, se convierte en una herramienta muy eficaz para colaborar y documentar.



#### **Ventajas de Usar Markdown en GitHub**

1. Lectura Clara en Cualquier Entorno



Los archivos siguen siendo comprensibles incluso en texto plano, pero GitHub los muestra con un formato limpio y atractivo de manera automática.



2\. Rapidez y Simplicidad



Es más ágil redactar documentación en Markdown que en HTML o en editores visuales, lo que agiliza el trabajo.



3\. Ideal para el Control de Versiones



Al ser texto plano, Git puede manejar los cambios con gran precisión, mostrando diferencias de forma clara.



4\. Funciones Extra de GFM



Permite usar referencias a issues y pull requests (como #123), algo muy útil para la organización dentro del repositorio.



Desventajas de Usar Markdown en GitHub

1\. Capacidades Limitadas



Markdown es intencionalmente simple. Si necesitas diseños muy elaborados, scripts o elementos interactivos, será necesario combinarlo con HTML o herramientas más avanzadas.



2\. Diferencias Entre Implementaciones



Cada plataforma puede interpretar Markdown de forma ligeramente distinta, y lo que se ve bien en GitHub podría mostrarse diferente en otros sitios.



3\. Dependencia de la Plataforma



El formato más atractivo se aprecia directamente en GitHub. Fuera de la plataforma, el archivo puede mostrarse solo como texto plano con la sintaxis visible.
