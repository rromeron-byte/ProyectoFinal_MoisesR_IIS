# ***Mantenimiento Propuesto para el Sistema de Control de Asistencia***

## ***1. Problema: Seguridad Vaga y Riesgosa***



El sistema de acceso es insuficiente, requiriendo solo una contraseña, lo que genera una debilidad que podría llevar a fallas de seguridad críticas.



***Tipo de Mantenimiento:*** Mantenimiento Preventivo



***Acción Principal:*** Implementar políticas de contraseñas robustas y gestión segura de sesiones.



***Justificación Técnica:*** Se aplica para prevenir futuros problemas. Fortalece el sistema y reduce los costos de mantenimiento a largo plazo.



***Tipo de Mantenimiento:*** Mantenimiento Perfectivo



***Acción Principal:*** Añadir Autenticación Multifactor (2FA).



***Justificación Técnica:*** Añade funciones que optimizan el uso (seguridad) y busca producir un sistema más mantenible.



#### ***2. Problema: Reportes Tarde y Poco Útiles para la Intervención***



El sistema solo genera un reporte simple de porcentaje de asistencia al final del semestre, lo que no permite a los profesores intervenir a tiempo.



***Tipo de Mantenimiento:*** Mantenimiento Perfectivo



***Acción Principal:*** Agregar un Módulo de Reportes de Alerta de Ausencias Frecuentes (reportes intermedios).



***Justificación Técnica:*** Es una adición de funcionalidad requerida por la variación en los requerimientos del proceso empresarial (necesidad de intervención temprana).

El sistema se optimiza para ser más útil para los profesores.
